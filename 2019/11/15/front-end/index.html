error openssl assert.h: No such file or directory #include <assert.h>

assert.h does exist in ./sysroot/usr/include/assert.h, but is not found by gcc.
maybe caused by android-ndk-r16b

The Android NDK has changed to Unified Headers:
https://android.googlesource.com/platform/ndk.git/+/master/docs/UnifiedHeaders.md

In r16b there now only a single set of system headers that is used for all API levels.

try android-ndk-r14b succeed for arm
when build for arm64 fail

最终方案：参考https://stackoverflow.com/questions/52717228/how-to-compile-openssl-1-1-1-for-android

Error: Permission denied @ apply2files - /usr/local/lib/node_modules/hexo/node_modules/extglob/lib/.DS_Store
brew doctor ，将warning解决完， 依然出错

解决方案

Error: EACCES: permission denied, mkdir ‘/usr/local/lib/node_modules/node-sass/build’
Ask Question
Solution :-

You need to change ownership of node_modules, because you use sudo npm install -g node-sass so its ownership set to root,

To change ownership to current user as group -

sudo chown -R root:YOUR_USERNAME /usr/local/lib/node_modules/

sudo chmod -R 775 /usr/local/lib/node_modules/
And never use sudo for npm related query.

rebuild， 根据报错，需引入静态库

复制到/cube/libs/armeabi及/cube/libs/armeabi-v7a

Link libuv to Android project error:src/unix/linux-core.c:508: error: undefined reference to ‘getpagesize’
reason: The minimum required platform version is android-21
solution: change minSdkVersion to 21
参考https://github.com/libuv/libuv/pull/633和
https://github.com/libuv/libuv/pull/633/commits/9bc3ecdc40ba331b8af382e9a1f239e44befbcd5

又出现报错：src/unix/udp.c:411: error: undefined reference to ‘in6addr_any’
参考https://github.com/libuv/libuv/pull/1433
最后将minSdkVersion调到24

error: undefined reference to ‘icu_64::UnicodeString::UnicodeString()’
参考：https://stackoverflow.com/questions/35246695/undefined-reference-to-icu-56unicodestringunicodestringsigned-char-unsign/35600138
这是由于查找函数时附上版本号造成
编译时加上–disable-renaming， 修改 unicode/uconfig.h， 并在使用库的代码中include unicode/uconfig.h
但依然报错：error: undefined reference to ‘icu::UnicodeString::UnicodeString()’
去掉前缀依然报错
参考readme.html ，修改uversion.h 依然报错
改成icu 56的版本，依然报错
怀疑是icu编译的两个静态库之间相互有依赖，库顺序导致问题，更改顺序后
尝试用cAPI, 避开析构函数，结果其中一个函数找不到undefined references
实在解决不了这个库的报错，怀疑在android上的兼容性有问题
最后选择用比较粗暴的方式代替了库的功能，将uint16_t与uint8_t字符串之间强行转换
